GENTRAIL / 02mental model
The one-sentence architecture

It is an evolutionary search wrapped around ordinary routing.

The app does not “invent” paths. It samples plausible loop shapes, asks A* to realize them on the mapped pedestrian network, then scores and de-duplicates the results.

01 / inputTrailhead + tastedistance, forest, trail, water, road aversion
02 / worldOSM snapshotOverpass features + walkable ways
03 / searchCandidate loopsA* over weighted graph; then mutate / crossover
04 / judgeScore + filtershape, nature, roads, repetition, variety
05 / output3–5 optionsmap lines, cards, warnings, GPX

The key design move: route generation and route evaluation are separate phases.

GENTRAIL / 10concurrency + UX
The main thread owns the conversation

Generation is cooperative; analysis is offloaded; cancellation is real.

main thread

Generate, report, render

  • Fetch Overpass data and build the graph.
  • Yield with a zero-timeout during long loops.
  • Stream phase progress from 0 → 100%.
  • Abort the active controller on cancel or new trailhead.
module worker

Analyze, score, explain

  • Repetition metrics and spatial indexing.
  • Route scoring with per-component explanations.
  • Progress messages and console logs cross the worker boundary.
  • Node fallback imports the same tasks for unit tests.
const controller = new AbortController();
currentController = controller;
try {
  const result = await generateHikes(settings, reportProgress, controller.signal);
} finally {
  worker.terminate("Generation finished");
}

GENTRAIL / 11reading the vibe
What rapid iteration left behind

It is surprisingly coherent — and the seams are visible in useful places.

strengths

Fast feedback became real structure

  • Feature modules are small and named after decisions.
  • Debug telemetry records timings, counts, fallbacks, and snap distance.
  • Unit tests cover A*, graph cleanup, crossover, metadata, and selection.
  • The score breakdown makes subjective heuristics legible.
scripts/test-unit.mjs · scripts/benchmark-gentrail.mjs
seams

Where an engineer should look twice

  • Heuristics are duplicated across traversal cost and scoring; behavior is tuned, not derived.
  • Random mutation plus Date.now() IDs make runs hard to reproduce exactly.
  • Public Overpass and CDN imports are runtime dependencies, not pinned data services.
  • “Fallback” is a first-class product path, which means strictness needs careful observability.
generateHikes.js · pathfinding.js · overpass.js

“Vibe-coded” here means an evolving system whose design is encoded in names, constants, fallbacks, and telemetry more than in a single formal spec.

GENTRAIL / 12how to change it safely
A practical handoff

Change one layer at a time; keep the diagnostics on.

The project is small enough to understand end-to-end. That is its superpower: the whole product loop is visible in a handful of files.

  • New OSM semantics? Start in osmQuery.js, overpass.js, and trailMetadata.js.
  • New route behavior? Touch graph costs, A*, or candidate creation in isolation.
  • New taste? Add a score component and its explanation before changing selection.
  • New guarantee? Add a unit test around the smallest pure module first.
  • New performance issue? Read the debug timings and benchmark the queue before guessing.
